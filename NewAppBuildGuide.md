# 作业登记与打分应用开发说明

## 项目定位

构建一个面向教师的本地作业登记与打分应用。应用以移动端使用为主，桌面浏览器可运行，Android 端通过 Capacitor 打包。

应用解决三件事：

- 快速记录一份作业中每名学生是否已交。
- 在需要时为学生录入分数和备注。
- 维护一份班级花名册，并让新建作业沿用当前名单。

第一版不包含账号、云同步、多班级、在线协作、复杂统计报表和服务端接口。所有业务数据保存在本机。

## 技术边界

- 前端使用原生 HTML、CSS、JavaScript ESM。
- 构建使用 esbuild。
- Android 使用 Capacitor。
- 数据保存在 `localStorage["asmc4_assignments_v1"]`。
- 应用语言为 `zh-CN`。
- 不引入大型前端框架，除非后续明确改变技术路线。

推荐目录结构：

```text
src/
  app/
    boot.js
    store.js
    actions/
    selectors/
  ui/
    render/
    panels/
    gestures/
  platform/
    backup.js
    native.js
  shared/
    constants.js
    normalize.js
    escapeHTML.js
    ids.js
  styles/
```

模块职责保持稳定：

- `boot.js` 只负责启动、绑定事件和首次渲染。
- `store.js` 只负责读取、保存、订阅和暴露状态。
- `actions/` 负责所有状态修改。
- `selectors/` 负责派生规则，比如统计、显示名、有效状态。
- `render/` 只读取状态并更新界面。
- `panels/` 管理抽屉、底部面板、确认框、设置页、花名册编辑器。
- `platform/` 处理浏览器和 Capacitor 差异。
- `shared/` 放常量、数据归一化、转义、id 生成等纯工具。

## 数据模型

持久化数据使用一个顶层对象：

```js
{
  hideNames: false,
  scoringMode: false,
  scoreTensMode: false,
  currentAssignmentId: "assignment-id",
  assignments: [
    {
      id: "assignment-id",
      title: "作业标题",
      subject: "英语",
      createdAt: "2026-06-25T00:00:00.000Z",
      updatedAt: "",
      students: [
        {
          id: 1,
          serial: "01",
          name: "学生姓名",
          status: "normal",
          badge: "",
          badgeType: "",
          note: "",
          updatedAt: ""
        }
      ]
    }
  ],
  roster: [
    {
      id: 1,
      serial: "01",
      name: "学生姓名",
      nonEnglish: false
    }
  ]
}
```

状态值：

```js
const STATUS = {
  NORMAL: "normal",
  REGISTERED: "registered",
  NONE: "none"
};
```

标记类型：

- `score` 表示分数。
- `submit` 表示提交标记。
- `note` 表示仅备注。
- `review` 表示已阅或其他只读标记。
- 空字符串表示无标记。

数据读取必须经过 `normalize`。旧字段、缺失字段、非法状态、空名单和损坏备份都要有兜底处理。学生 id 比较时统一转成字符串。

## 产品规则

### 作业

- 应用首次打开时创建一份默认作业和默认 50 人名单。
- 新建作业默认复制当前花名册。
- 新建作业时可填写标题，也可留空，留空时生成默认标题。
- 作业可设置科目，科目为空表示不指定。
- 作业列表显示每份作业的标题、科目、已交数、总人数和未交数。
- 删除作业前必须确认。
- 删除最后一份作业后自动创建一份空白作业，避免应用进入无作业状态。
- 当前作业可反选提交状态，无登记学生不参与反选。

### 学生状态

- 学生有三种状态：普通、已登记、无登记。
- 普通学生点击后变为已登记。
- 已登记学生点击后变为普通。
- 无登记学生不能通过点击改变状态。
- 学生卡片显示序号、姓名或匿名代号、分数或标记、备注。
- 已登记状态要有明确视觉反馈。
- 无登记状态要降低视觉权重。

### 隐藏姓名

- 开启隐藏姓名后，学生姓名显示为干支代号。
- 代号按学生在当前作业中的顺序生成。
- 关闭后恢复真实姓名。
- 无障碍文本也应使用当前显示名。

### 科目与非英语规则

- 花名册中的学生可以标记为“非英语”。
- 当作业科目为“英语”时，非英语学生的有效状态视为无登记。
- 这类学生不计入总人数、已交数和未交数。
- 这类学生不能通过点击、反选或打分改变状态。
- 判断逻辑集中放在 selector，不分散写在渲染和事件代码里。

### 打分

- 应用有全局打分模式。
- 打分模式关闭时，点击学生切换提交状态。
- 打分模式开启时，点击学生打开打分面板。
- 长按学生也可打开打分面板。
- 打分面板显示学生序号、显示名、分数、备注输入框和数字键盘。
- 分数为正整数时，保存后学生状态变为已登记，标记类型为 `score`。
- 分数为空或 0 时，保存后清空分数标记，学生状态变为普通。
- 备注单独保存到 `note`，不应被分数清空。
- `×10` 模式开启后，数字键 `1` 到 `9` 直接代表 `10` 到 `90`，`0` 代表 `100`，点击后立即保存。
- `×10` 模式是全局偏好，需持久化。

### 花名册

- 花名册可编辑姓名、添加学生、删除学生、拖拽排序。
- 至少保留一名学生。
- 保存花名册时，每名学生必须有姓名。
- 排序后重新生成两位序号。
- 可将学生标记为“非英语”。
- 重置花名册会恢复默认 50 人。
- 花名册是新建作业的来源。
- 已有作业的学生记录默认保持不变，避免历史作业被后续名单调整改写。

### 备份

- 支持导出 JSON 备份。
- 支持导入 JSON 备份。
- 导入前必须确认，因为会覆盖当前本地数据。
- 导入数据必须经过 normalize。
- 浏览器端导出为下载文件。
- Android 端导出到缓存目录后调用系统分享。
- 备份文件名包含年月日和时分秒。

## 界面结构

第一屏是实际工作界面，不做营销页或说明页。

主要区域：

- 顶部栏：菜单按钮、当前作业标题、设置按钮。
- 进度条：显示已交、总数、未交。
- 学生网格：展示全部学生卡片。
- 底部或悬浮操作：新建作业、打分模式、隐藏姓名等高频入口。
- 作业抽屉：列出作业，支持切换、编辑、删除、新建。
- 快捷面板：从标题进入，支持快速切换作业、改名、改科目。
- 打分面板：底部面板或居中面板，适配手机单手操作。
- 设置面板：打分模式、`×10` 模式、备份、花名册入口。
- 花名册编辑器：列表编辑、拖拽排序、保存、重置。
- 确认框：删除、导入、重置、反选等破坏性或批量操作。

视觉要求：

- 界面安静、紧凑、偏工具属性。
- 学生卡片尺寸稳定，状态变化不能造成网格跳动。
- 卡片、按钮和面板的圆角克制。
- 颜色用于表达状态，不用装饰性渐变和无意义背景图形。
- 移动端文字不能溢出按钮、卡片或面板。
- 当前作业、已登记、无登记、危险操作要有可区分状态。

## 交互规则

- 抽屉和普通面板不使用半透明遮罩。
- 抽屉和面板可通过关闭按钮、空白区域点击、空白区域滑动、`Escape` 或 Android 返回键关闭。
- 确认框打开时，遮罩点击不关闭下层面板。
- `Escape` 按当前层级关闭界面：确认框、打分面板、花名册、设置、普通面板、抽屉。
- Android 返回键按同样顺序关闭界面，全部关闭后退出应用。
- 面板切换期间要避免重复打开或关闭导致状态错乱。
- 支持触摸反馈和必要的轻震动。
- 长按打开打分面板后，应抑制随后触发的普通点击。

## 安全与可靠性

- 用户输入进入 `innerHTML` 前必须 `escapeHTML()`。
- 优先用 DOM API 和 `textContent` 写入用户文本。
- `localStorage` 写入失败时允许应用继续运行，并在控制台输出警告。
- 导入失败、JSON 解析失败和格式不合法时要给出明确提示。
- 状态修改后统一保存，再触发渲染。
- 渲染函数不直接修改业务状态。
- 业务 action 不直接操作 DOM。
- 新增持久化字段时同步 normalize、导入、导出和默认数据。

## 构建与运行

推荐命令：

```bash
node build.mjs
npm run preview
npm run cap:sync
npm run cap:open
npm run cap:run
```

构建输出目录为 `dist/`。Capacitor 的 `webDir` 指向 `dist`。

## 验收清单

基础流程：

- 首次打开应用能显示默认作业和默认学生。
- 新建作业后自动切换到新作业。
- 作业可切换、重命名、设置科目、删除。
- 删除最后一份作业后仍有可用作业。

提交登记：

- 点击普通学生后变为已登记。
- 再次点击已登记学生后变为普通。
- 反选只影响可登记学生。
- 进度统计与卡片状态一致。

打分：

- 打分模式开启后点击学生打开打分面板。
- 长按学生打开打分面板。
- 保存分数后学生显示分数并计为已登记。
- 清空分数后学生回到普通状态。
- 备注保存后显示在学生卡片上。
- `×10` 模式能直接录入 10 到 100。

花名册：

- 可添加、删除、改名、排序学生。
- 空姓名不能保存。
- 至少保留一名学生。
- 新建作业使用保存后的花名册。
- 英语作业中非英语学生不计入统计且不能操作。

备份：

- 导出文件内容包含作业、花名册和偏好设置。
- 导入有效备份后界面更新。
- 导入非法文件时不覆盖当前数据。

移动端：

- 抽屉、面板、打分页在手机视口下不遮挡关键按钮。
- Android 返回键关闭当前最上层界面。
- 无打开面板时返回键退出应用。

构建：

```bash
npm run build
```

构建必须通过。

## 开发约束

- 功能完成顺序以数据模型、状态 action、渲染、交互、平台能力为主。
- 不在开发中途改变存储 key。
- 不把平台能力写进通用业务模块。
- 不让事件处理函数直接散落复杂业务判断。
- 不把统计逻辑复制到多个渲染函数。
- 不在第一版加入未列入本文的业务范围。
